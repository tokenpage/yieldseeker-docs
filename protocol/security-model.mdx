# Security Model

Yieldseeker uses a defence-in-depth security model.

Rather than relying on a single protection mechanism, security is achieved through multiple independent layers that work together to reduce risk and constrain execution.

Each layer protects a different part of the system, helping limit the impact of individual failures.

---

## User Ownership

Users remain the owners of their assets at all times.

Each Agent operates through its own isolated Agent Wallet, and only the wallet owner can withdraw assets.

Portfolio management may be autonomous, but ownership always remains with the user.

---

## Constrained Execution

The Allocation Engine cannot execute arbitrary blockchain transactions.

Every action must pass through the Execution Framework, where it is validated against protocol-defined rules before execution.

This significantly limits the set of actions an Agent can perform.

---

## Layered Validation

Multiple validation layers work together before every transaction.

These include:

- Adapter Registry allowlists
- protocol-specific adapters
- parameter validation
- protocol compatibility checks

Transactions that fail validation are rejected before execution.

---

## Infrastructure Security

Yieldseeker uses secure infrastructure to protect execution credentials.

This includes:

- Coinbase Trusted Execution Environments (TEE)
- encrypted key management
- hardware-backed execution

Sensitive cryptographic operations occur within isolated hardware environments rather than general-purpose infrastructure.

---

## Administrative Security

Protocol upgrades are protected through governance controls.

Administrative actions require:

- hardware-backed multisignature approval
- a four-day timelock before activation

This provides transparency and allows users to review changes before they take effect.

---

## Operational Security

The protocol continuously monitors supported integrations and execution infrastructure.

Operational controls include:

- continuous protocol monitoring
- emergency adapter pause capabilities
- ongoing infrastructure maintenance
- incident response procedures

These controls help reduce operational risk while allowing the protocol to evolve over time.

---

## Audits

Yieldseeker's smart contracts undergo independent security reviews before deployment.

Audit reports are published publicly as they become available.

Security reviews complement—but do not replace—the protocol's layered security architecture.

---

## Security Through Separation

The protocol deliberately separates intelligence, decision-making, and execution into independent subsystems.

- **Agent Intelligence** understands user intent.
- The **Allocation Engine** determines portfolio allocations.
- The **Execution Framework** validates and executes transactions.

This separation reduces trust assumptions, limits system complexity, and allows each layer to evolve independently while preserving the protocol's overall security guarantees.
